# 通配符
## ？

![](assets/文件名过滤/file-20260109114858618.png)

?表示代替任一字符
所以会匹配任何符合fl?g.php形式的文件

![](assets/文件名过滤/file-20260109114929137.png)

甚至只需要相同的格式即可

![](assets/文件名过滤/file-20260109114949765.png)

## *
星号表示从当前位后面全部模糊匹配
例如：
cat /f* 就是匹配所有f开头的文件

![](assets/文件名过滤/file-20260109115108487.png)

# 引号' & "
单引号''或者双引号""
用以包裹空字符串，从而让系统识别为空，连接上前后的字符组成完整的命令
同时没有出现完整的过滤字符串，从而绕过

![](assets/文件名过滤/file-20260109115149033.png)

![](assets/文件名过滤/file-20260109115152901.png)

在passthru()包裹的指令里面已经被""包裹，因此绕过需要用单引号''
反之外层为单引号里层绕过就用双引号即可

# 反斜线
\可以过滤掉一些特殊含义字符的功能，仅仅输出字符的作用，例如某些情况$%#等符号以及连接其他字符有特殊含义和作用，使用\即可仅仅输出字符的功能
因此和C语言类似，为转义字符

例如上文提到的tac fl?g.php flag*.php
此时在?和*前面加上转义字符\则会实现精准匹配

![](assets/文件名过滤/file-20260109115248065.png)

# 内联执行
给不同的变量赋值最后组合成完整的命令
例如：
```php
a=fl;b=ag;c=ph;d=p;cat $a$b.$c$d
```

![](assets/文件名过滤/file-20260109115512658.png)

通过给某些变量定义后，使用；连接命令即可执行完整命令

![](assets/文件名过滤/file-20260109115519818.png)
